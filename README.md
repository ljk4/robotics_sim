## 3. 正向运动学 FK

### 3.1 问题描述

给定关节角度向量 **q** ∈ R^9，求解末端执行器的世界坐标 **p_ee** ∈ R^3。

### 3.2 方法一：MuJoCo 内部计算

调用 `mj_forward()` 计算全部连杆的世界位姿，直接读取末端位置：

```python
data.qpos[...] = q
mujoco.mj_forward(model, data)
p_ee = data.xpos[body_id] + data.xmat[body_id] @ offset_local
```

### 3.3 方法二：解析变换矩阵连乘

从骨盆到末端，逐连杆计算 4x4 齐次变换矩阵之积：

```
T_world_ee = T_world_pelvis * T_torso * T_shoulder_pitch * ... * T_elbow

其中 T_parent_child = | R_quat * R_joint(q)   pos_body |
                       | 0      0      0         1     |
```

**MuJoCo 运动学关键公式（经实验严格验证）：**

```
xpos[child]  = xpos[parent] + xmat[parent] * pos_body
xmat[child]  = xmat[parent] * R_quat(quat_child) * R_joint(q)
                                                        ^^^^^^^^
                                    注意: 先 body quat, 后 joint rotation!
```

该旋转顺序是经过严格对比测试验证的关键发现。错误的顺序 (`R_joint * R_quat`) 会导致旋转矩阵误差达 0.7，末端位置误差 > 0.5m。

### 3.4 FK 验证

50 组随机关节角度对比：

| 方法 | 误差 |
|------|------|
| MuJoCo FK vs 解析法 | 最大 **8.18e-15 m** |
| 结论 | 完全一致（数值精度极限） |

## 4. 逆向运动学 IK

### 4.1 问题描述

给定末端目标世界坐标 **p_target**，求解最优关节角度：

```
min  || FK(q) - p_target ||^2
s.t. q_lower <= q <= q_upper
```

### 4.2 方法一：雅可比阻尼最小二乘法 DLS

**核心迭代公式：**

```
q_{k+1} = q_k + J^T (J J^T + λ^2 I)^{-1} e
```

- **J** ∈ R^{3×n}：末端执行器的点雅可比矩阵
- **e** = p_target - p_current：位置误差
- **λ**：阻尼因子（防止奇异位形下发散）

**点雅可比计算（含偏移修正）：**

末端执行器不在连杆原点（offset = [0.28, 0, -0.015]m），必须修正雅可比：

```
J_point = J_trans - skew(r_world) * J_rot

r_world = xmat[body] @ offset_local
skew(r) = |  0   -rz   ry |
          |  rz   0   -rx |
          | -ry   rx   0  |
```

若不修正此偏移，仅用连杆原点的平移雅可比（忽略旋转分量），IK 误差可达 > 50cm。这是通过实际调试发现的关键问题。

**特点：** 收敛快（~2ms），适合实时控制；但为局部方法，可能收敛到"背后解"等局部最优。

### 4.3 方法二：数值优化法 L-BFGS-B + 正则化

**求解策略：**

1. **优化算法**：L-BFGS-B（有限内存拟牛顿法，支持边界约束）
2. **全局搜索**：12 起点（含零起点 + 11 个随机起点）
3. **正则化代价函数**：

```
cost(q) = ||FK(q) - p_target||^2 + α * ||q||^2
```

其中 **α = 1e-4** 为正则化权重。该正则化项是关键：

**正则化的必要性（实际发现）：**

5 DOF（躯干+4臂关节）对 3D 定位有 2 个冗余自由度。不加正则化时，优化器可能找到"怪异解"——如躯干翻转 133°（q_torso ≈ -2.3 rad），手臂以扭曲姿态补偿，虽 FK 误差极小（~1e-13 m²），但关节角度远离自然姿态。PD 控制器跟踪此类轨迹时，末端的中间路径偏离目标可达 600mm。

加入正则化后（α=1e-4），优化器偏向零姿态附近的自然解：
- 零姿态 FK 误差约 4.2mm → cost ≈ 1.8e-5
- 翻转解正则化罚项 ≈ 1e-4 × 5.5 ≈ 5.5e-4 > 1.8e-5 → **零姿态胜出**
- 正常前伸所需的角度（≈0.3rad）罚项 ≈ 1e-5 → **不影响正常调节**

**双臂独立性：** 仿真演示中默认 `use_torso=False`。4 DOF 手臂对 3D 定位已足够（1 冗余），且左右臂独立求解避免了共享躯干关节的冲突问题。

**特点：** 鲁棒性强（成功率 100%），但较慢（~100ms），适合离线轨迹规划。

### 4.4 IK 验证结果

| 测试项 | 雅可比 DLS | 数值优化 L-BFGS-B |
|--------|-----------|-------------------|
| 100 组随机目标 | 成功率 99% | 成功率 **100%** |
| 平均跟踪误差 | 0.07 mm | 0.07 mm |
| 单次求解耗时 | ~2 ms | ~100 ms |
| 前伸目标鲁棒性 | 偶尔局部最优 | 始终正确（正则化保证） |

## 5. 动力学

### 5.1 运动方程

```
M(q) * qddot + C(q, qdot) + G(q) = tau
```

| 符号 | 含义 | 维度 |
|------|------|------|
| M(q) | 质量矩阵（对称正定） | 9×9 |
| C(q,qdot) | 科氏力 + 离心力 + 关节阻尼 | 9 |
| G(q) | 重力力矩 | 9 |
| tau | 关节驱动力矩 | 9 |
| qddot | 关节加速度 | 9 |

### 5.2 质量矩阵 M(q)

通过 `mj_fullM()` 提取稠密质量矩阵：

| 属性 | 数值 | 说明 |
|------|------|------|
| 条件数（零姿态） | 6.1 | 非常良态 |
| 条件数范围 | [6.6, 15.7] | 全域良态，与姿态相关 |
| 对称性 | 误差 0.00 | 精确对称 |
| 最小特征值 | 0.11 > 0 | 严格正定 |
| 非对角耦合占比 | < 10% | H1 上肢连杆间耦合较弱 |

### 5.3 逆动力学 ID

**问题：** 给定运动 (q, qdot, qddot)，求所需力矩 tau。

```python
# 方法一: MuJoCo mj_inverse
data.qacc[:] = qddot
mujoco.mj_inverse(model, data)
tau = data.qfrc_inverse[...]

# 方法二: 手动公式 (M * qddot + bias)
bias = mj_inverse(qddot=0)       # = C(q,qdot) + G(q) + damping
tau = M @ qddot + bias
```

**关键实现细节：** 需先调用 `mj_forward` 更新 `data.qM`，再调用 `mj_fullM` 提取 M。bias 通过 `mj_inverse(qacc=0)` 获取，与 ID 内部计算一致。

**验证：** 双方法误差 < 2.22e-16 N·m（数值精度极限）

### 5.4 正动力学 FD

**问题：** 给定力矩 tau 和状态 (q, qdot)，求加速度 qddot。

```python
# 方法一: MuJoCo mj_forward
data.qfrc_applied[:] = tau
mujoco.mj_forward(model, data)
qddot = data.qacc[...]

# 方法二: 手动公式
qddot = M^{-1} @ (tau - bias)
```

**验证：** 双方法误差 < 5.68e-14 rad/s^2（数值精度极限）

### 5.5 重力分析

零姿态下各关节的重力力矩分布：

| 关节 | 重力力矩 (N·m) | 原因 |
|------|---------------|------|
| torso | 0.000 | Z 轴旋转不受重力影响 |
| shoulder_pitch | -1.097 | 手臂自然下垂产生 |
| shoulder_roll | ±0.031 | 对称位置近似为零 |
| shoulder_yaw | ≈0 | 同 shoulder_roll |
| elbow | -1.044 | 前臂质量产生 |

躯干绕 Z 轴旋转不改变重力在关节空间的方向 → 重力力矩在各躯干角度下恒为常量。

### 5.6 科氏力分析

同一姿态、不同速度下的科氏力贡献：

| 速度 (rad/s) | 重力 norm | 科氏力 norm | 科氏力占比 |
|-------------|----------|------------|----------|
| 0.5 | 2.14 | 0.23 | 11% |
| 1.0 | 2.14 | 0.91 | 48% |
| 2.0 | 2.14 | **3.65** | **111%** |

**结论：** 高速运动（qdot ≥ 1 rad/s）时科氏力不可忽略，必须纳入动力学模型或通过 PD 控制器的高增益来压制。

### 5.7 PD + 重力前馈控制器

```
tau = Kp * (q_des - q) + Kd * (qd_des - qd) + G(q)
       |____比例项____|     |____阻尼项____|     |_前馈_|
```

| 项 | 物理含义 | 作用 |
|----|---------|------|
| Kp·Δq | 弹簧恢复力 | 驱动关节朝向目标 |
| Kd·Δqd | 粘性阻尼 | 抑制震荡，Kd ≈ Kp/10 |
| G(q) | 重力前馈 | 精确抵消重力，消除稳态误差 |

**重力补偿对比：**

| 配置 | 稳态关节误差 |
|------|------------|
| 带重力补偿 | ~0 mrad |
| 无重力补偿 | e_ss = G(q)/Kp ≈ 2~7 mrad |

仿真中按 `G` 键可实时切换，直观观察重力补偿的效果。

### 5.8 力矩分解

逆动力学力矩可分解为三个物理分量：

```
tau = M(q)*qddot  +  C(q,qdot)  +  G(q)
      |_惯性项_|    |_科氏项_|    |_重力项_|
```

在仿真演示的终端显示中，`|tau|` 为实际施加力矩、`|G|` 为重力补偿力矩、`|qdd|` 为当前加速度。

### 5.9 ID → FD 闭环验证

```
原始加速度 qddot_original
  → ID: tau = ID(q, qdot, qddot_original)
  → FD: qddot_recovered = FD(q, qdot, tau)
  → 比较: ||qddot_recovered - qddot_original||

结果: 最大误差 < 9e-16 rad/s^2 —— 闭环完全一致
```

按 `I` 键可在仿真运行中随时触发此验证。

## 6. 仿真演示

`simulation_demo.py` 提供交互式 MuJoCo 3D 仿真演示，涵盖运动学和动力学的全部核心内容。

### 6.1 演示功能

**运动学部分：**
- **IK 预求解**：6 组目标姿态，L-BFGS-B + 正则化求解关节角度
- **FK 实时验证**：终端显示左右末端执行器与目标的实时误差（mm）

**动力学部分：**
- **PD+重力前馈控制**：实时力矩显示（|tau|, |G|, |qdd|）
- **重力补偿对比**：按 `G` 键切换 ON/OFF，观察稳态误差变化
- **ID/FD 验证**：按 `I` 键在仿真中运行双方法验证 + 质量矩阵属性
- **关节误差显示**：实时 |q_err| 显示关节空间跟踪精度（mrad）

### 6.2 演示流程

```
Home → 右臂上举 → 左臂前伸 → 双臂前伸 → T型平举 → 不对称姿态 → 循环
```

每个姿态的步骤：
1. **IK 预求解**（仿真启动时完成）→ 得到目标关节角度 q_goal
2. **轨迹生成**（五阶多项式 smooth step）→ 位置/速度/加速度均连续
3. **PD+重力前馈控制** → 关节空间轨迹跟踪，力矩限幅 ±150 Nm
4. **FK 实时验证** → 终端 0.5s 间隔显示末端误差和动力学数据

### 6.3 轨迹生成：五阶多项式

```
s(t) = 10t^3 - 15t^4 + 6t^5            (t ∈ [0, 1])

边界条件:
  s(0)=0,   s(1)=1      位置连续
  s'(0)=0,  s'(1)=0     速度连续（零启停，无冲击）
  s''(0)=0, s''(1)=0    加速度连续（力矩平滑）

轨迹映射:
  q_des(t)  = q_start + s(t/T) * (q_goal - q_start)
  qd_des(t) = (q_goal - q_start) * s'(t/T) / T
```

与线性插值的对比：
- 线性插值：速度在起止点突变 → 需要无穷大加速度/力矩
- 五阶多项式：所有边界条件平滑 → 电机力矩连续变化，实际可执行

### 6.4 终端显示格式

```
[跟踪] 4-双臂前伸  进度: 45.0%  左EE:   3mm  右EE:   5mm  |q_err|=   12mrad  Grav:ON  |  |tau|= 12.3  |G|=  5.1  |qdd|=  2.8
```

| 字段 | 含义 |
|------|------|
| `[跟踪/停留]` | 状态机阶段 |
| `进度` | 轨迹完成百分比 |
| `左EE/右EE` | FK 末端位置与目标偏差 (mm) |
| `|q_err|` | 当前关节角与目标角的 L2 误差 (mrad) |
| `Grav` | 重力补偿 ON/OFF |
| `|tau|` | 实际施加力矩 L2 范数 (Nm) |
| `|G|` | 重力补偿力矩 L2 范数 (Nm) |
| `|qdd|` | 关节加速度 L2 范数 (rad/s²) |

### 8.3 关键技术发现

1. **MuJoCo 旋转顺序**：`xmat[child] = xmat[parent] * R_body_quat * R_joint`（先 body quat 后 joint rotation）。此顺序经严格实验验证，错误顺序导致旋转误差 0.7。

2. **点雅可比偏移修正**：末端执行器离连杆原点 0.28m，必须修正雅可比 `J_point = J_trans - skew(r) * J_rot`。忽略此修正导致 IK 误差 > 50cm。

3. **IK 正则化必要性**：冗余自由度下优化器可能找到"怪异解"（躯干翻转+手臂扭曲）。正则化 `α=1e-4·||q||^2` 偏向自然姿态，权重精心选取以允许正常调节而阻止翻转。

4. **雅可比 DLS vs L-BFGS-B**：前者快速（2ms）适合实时控制；后者鲁棒（100ms）适合离线规划。实际使用中互补。

5. **重力补偿必要性**：纯 PD 控制有固有稳态误差 `e_ss = G(q)/Kp`，重力前馈可消除。仿真按 `G` 键实时对比。

6. **科氏力**：高速运动（qdot=2 rad/s）时科氏力可达重力的 1.1 倍，不可忽略。

7. **质量矩阵状态依赖**：`mj_fullM` 依赖 `data.qM`，需先调用 `mj_forward` 更新。在多步动力学计算中必须先 forward 再提取 M。
